# Contributing to kavosh-backend
Thank you for considering contributing to kavosh-backend! We appreciate your help in making our project the best it can be.

## Forking the Repository
To contribute to kavosh-backend, follow these steps:
1. **Fork the repository**: Click the "Fork" button at the top-right corner of the [kavosh-backend repository](https://github.com/your-username/kavosh-backend) to create a copy of the repository in your own GitHub account.
2. **Clone the repository**: Run `git clone https://github.com/your-username/kavosh-backend.git` to clone the forked repository to your local machine.
3. **Create a new branch**: Run `git checkout -b your-branch-name` to create a new branch for your changes.

## Branch Naming
When creating a new branch, use the following naming conventions:
* **Feature branches**: `feature/your-feature-name`
* **Bugfix branches**: `fix/your-bugfix-name`
* **Refactor branches**: `refactor/your-refactor-name`

## Commit Style
When making changes, follow these guidelines for commit messages:
* Use the imperative mood (e.g., "Add new feature" instead of "Added new feature")
* Keep the first line concise (<50 characters) and focused on the change
* Use a blank line to separate the brief summary from the body
* Use bullet points in the body to explain the changes

Example:
```
Add new endpoint for user authentication

* Create new API endpoint for user login
* Update user model to include authentication details
```

## Pull Request Template
When submitting a pull request, use the following template:
```markdown
### Description
 Briefly describe the changes made in this pull request.

### Changes
* List the specific changes made in this pull request.

### Related Issues
* List any related issues or bugs fixed in this pull request.

### Testing
* Describe how the changes were tested and verified.
```

## Code of Conduct
We are committed to providing a welcoming and inclusive environment for all contributors. Please review our [Code of Conduct](https://github.com/your-username/kavosh-backend/blob/main/CODE_OF_CONDUCT.md) before contributing to kavosh-backend.

Thank you again for your contribution! We look forward to reviewing your pull request.
